# Synthesis Changelog

## Consensus

- The next iteration should improve visible TUI power first, because users can immediately benefit from richer graph navigation and command discovery.
- Plugin SDK should continue, but the next valuable step is adapter execution plus host-mediated commit, not marketplace or custom renderers.
- Triage should become a workflow queue rather than only a ranked list.

## Resolved Conflicts

- [RESOLVED] Broad Labs vs focused next slice: implement Graph Lab first, keep Trace/Diff/Crash/Firmware as later lenses or adapter-backed data sources.
- [RESOLVED] Custom plugin UI vs host-owned templates: use declarative slots and existing TUI surfaces first.
- [RESOLVED] Scoring extensibility vs trust: defer rule-pack SDK until plugin provenance and contribution commit are stable.

## Suggested Milestones

- M2A TUI Power Navigation: F-001, F-003, F-004, F-005, F-008.
- M2B Adapter Commit Pipeline: F-002, F-007.
- M3 Triage Intelligence: F-006.
